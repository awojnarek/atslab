OS Metrics Collector is a project born out of the following principles:
+ Remove the idea of "collect it messy and throw it over the ETL fence"
+ Three P's: Powerful, Performant and Pluggable
+ A single codebase that provides native binaries for (x86/Power/ARM):
- Linux
- AIX
- Windows
- MacOS
- OpenBSD
- Solaris
+ No reliance on outside binaries. The only current outside libraries used are:
- gopsutil (provide basic cpu/memory/disk/network metrics cross platform)
- viper (provide configurations)
- cobra (provide command-line flags / organization)
+ Innovative features (gap fillers):
- "Near real-time" collection
- ASCII historical graphs on CLI
- Immediate and instantaneous Galileo support
- Node-level Zabbix/Service Now support
